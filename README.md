# 🚀 自動レポート生成システム・アップデートまとめ（2025/11/29）

このドキュメントは、2025/11/29 に実施した大規模改善内容をまとめたものです。  
Slack Block Kit 化、ログ強化、メール送信整理、main処理リファクタリングなど、  
本プロジェクトを “実務品質の自動レポート生成システム” に進化させた内容です。

------------------------------------------------------------
📦 変更点まとめ（すべて本日実施）
------------------------------------------------------------

【1. プロジェクト全体の構造改善】
- .env の読み込み位置調整
- Slack／ログ／メール依存関係の整理
- logger_setup.py に重複ハンドラ防止ロジック追加
- main処理へ try/except 統合 → 致命的エラーを確実に捕捉

------------------------------------------------------------
【2. Slack通知の Block Kit 化】
------------------------------------------------------------
- send_slack(title, color, details) 形式に統一
- 成功／警告／致命的エラーで色・絵文字が自動で変わる
- Webhook 未設定時は安全にスキップ
- 運用しやすく視認性の高い通知に進化

（通知例）
🟩 成功：color="good"
⚠️ 失敗：color="warning"
🔥 致命的エラー：color="danger"

------------------------------------------------------------
【3. ログ強化（logger_setup.py）】
------------------------------------------------------------
- TimedRotatingFileHandler による日次ローテーション
- auto_report.log を固定名にし、古い分は自動で日付付与
- ログフォーマット統一：
  2025-11-29 15:01:23 | INFO | auto_report | run_script | 開始

- ハンドラ重複登録の防止（安定性UP）

------------------------------------------------------------
【4. main() の処理フローリファクタリング】
------------------------------------------------------------
＜新しいフロー＞
1. ログ開始  
2. 子スクリプト実行（sales_total / high_sales / generate_index）  
3. 成功 or 失敗の Slack Block Kit 通知  
4. 処理時間ログ  
5. メール送信  
6. 想定外例外は Slack danger＋ログで通知  

（成功）
🟩 全スクリプト正常完了！

（失敗）
⚠️ 一部スクリプトに失敗あり

（致命的エラー）
🔥 メイン処理で致命的エラー

------------------------------------------------------------
【5. メール送信処理の整理】
------------------------------------------------------------
- .env の sender/receiver を一元管理
- エラー時もログに残してクラッシュ防止
- 添付ファイル：
  outputs/sales_chart.png
  outputs/top_sales_plot.html

------------------------------------------------------------
【6. 本日は実行テストは未実施（構造整理フェーズ）】
------------------------------------------------------------
- 今日の作業は「コード統合」「構造整理」「通知＆ログ整備」
- 実行テストは明日でOK（構造は完成済み）
