# 🧩 自動レポート改善システム・アップデートまとめ（2025/11/29）

このドキュメントは、2025/11/29 に実施した大規模改善内容をまとめたものです。
Slack Block Kit 対応、ログ強化、メール送信処理、main処理リファクタリングなど、
本プロジェクトを「単なるスクリプト」から「売上レポート生成システム」に進化させた内容です。

------------------------------------------------------------
[1. プロジェクト全体の巻き直し]
- .env の整理、Slack 送信ロジック追加
- Slack メッセージ・例外処理の整備
- logger_setup.py にローテーション／重複防止ロジック追加
- main処理の try/except 整備 → すべてのエラー種別を捕捉

[2. Slack通知の Block Kit 化]
- send_slack(title, color, details) 形式に統一
- 成功／警告／致命的エラー で色・絵文字が自動で変わる
- Webhook 未設定時は安全にスキップ
- 運用しやすく情報量の高い通知へ進化

《通知例》
🟢 成功：color="good"
🟡 警告：color="warning"
🔴 致命的エラー：color="danger"

------------------------------------------------------------
[3. ログ強化（logger_setup.py）]
- TimedRotatingFileHandler による日次ローテーション
- auto_report.log を固定名にし、古い分は自動で日付付与
- ログフォーマット統一：
  2025-12-29 15:01:23 | INFO | auto_report | run_script | 開始
- ハンドラ重複登録の防止（安定性UP）

------------------------------------------------------------
[4. main() の処理フローリファクタリング]

《新しいフロー》
1. ログ開始  
2. 各スクリプト実行（sales_total / high_sales / generate_index）  
3. 成功 or 失敗の Slack Block Kit 通知  
4. 処理時間計測  
5. メール送信  
6. 想定外例外は Slack danger で通知  

《成功》
🟢 全スクリプト正常完了！

《失敗》
⚠️ 一部スクリプトが失敗あり

《致命的エラー》
🔥 メイン処理で致命的エラー

------------------------------------------------------------
[5. メール送信処理の整理]
- .env の sender/receiver を一元管理
- エラー時もログに残してクラッシュ防止
- 添付ファイル：
  outputs/sales_chart.png
  outputs/top_sales_plot.html

------------------------------------------------------------
[6. 本日は実行テストは未実施（整備整理フェーズ）]
- 今日の作業は「コード統合」「構造整備」「通知＆ログ整備」
- 実行テストは明日でOK（構造は完成済み）

------------------------------------------------------------
[7. 更新したファイル一覧]
auto_sales_report.py …… Slack Block Kit / main統合 / ログ統合  
logger_setup.py …… ローテション／重複防止／統一フォーマット  
.env …… 読み込み整理  
send_mail.py …… mainとの連携整理  

------------------------------------------------------------
[8. 次のステップ]
